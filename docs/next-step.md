# Next Step

## Phase 4 目标
- 只优化主流程中必要的交互反馈和视觉层级。
- 不改业务规则，不扩展功能范围，不新增组件。

## 本轮任务顺序
1. 收敛 `scoreboard` 的信息层级与分数状态反馈。
2. 收敛 `round-entry` 的页面节奏、提示区和主操作区阅读顺序。
3. 收敛 `round-confirm` 的汇总区与确认操作区层级。
4. 收敛 `hu-form` 的分组感、输入区可读性和预览反馈。
5. 必要时只在 `app.less` 中补最小的全局样式基线。

## 验收标准

### 主流程页面
- 结果录入页更容易看出：
  - 当前局状态
  - 本局实时分
  - 主要操作入口
  - 血流成河提示信息
- 本局确认页更容易看出：
  - 本局合计
  - 结算记录
  - 返回修改 / 确认累计

### 核心组件
- 记分板中累计分与本局变化的层级更清楚。
- 正分、负分、零分的状态反馈更稳定。
- `hu-form` 中不同输入组与预览区更容易区分。

### 工程约束
- 不修改核心业务逻辑，除非样式接线必须依赖极小调整。
- 不改统一重算路径。
- 不重构无关文件。
- diff 保持小且可审查。

## 当前注意事项
- 继续保留首页为独立 page。
- `hu-form` 继续作为 `round-entry` 页内 modal。
- 不开始大规模视觉重做，只做必要 refinement。
- 若后续要扩展 Phase 4，优先补状态反馈，再考虑更细的视觉统一。
