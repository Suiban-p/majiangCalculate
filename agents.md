# AGENTS.md

## 1. 项目类型

本项目为 **微信原生小程序 + TypeScript**。

技术约束：

- 使用原生微信小程序目录结构
- 使用 `.wxml / .wxss / .ts / .json`
- 不使用 Vue / React / Taro / uni-app
- 不使用浏览器 DOM API
- 不使用 web-only 组件和样式方案
- 最终代码必须可在微信开发者工具中直接运行

---

## 2. 项目目标

根据 `plain.md` PRD 和 Stitch UI 设计，实现：

四川麻将算番小程序 V1.0

核心目标：

- 支持血战到底完整流程
- 支持血流成河基础流程
- 支持实时累计记分
- 支持单局多次结算
- 支持本地持久化
- 支持撤销上一步
- 支持一炮多响

---

- `plain.md` 是产品需求唯一事实来源

## 10. 业务规则

### 10.1 优先实现血战到底

要求：

- 胡牌后玩家退出后续胡牌选择
- 支持 1 到 3 次胡牌
- 剩余人数小于等于 1 时自动结束本局
- 支持杠牌
- 支持查大叫

### 10.2 血流成河（V1）

只实现：

- 多次胡牌
- 一炮多响
- 手动结束本局

暂不实现：

- 牌墙自动计数
- 复杂计时逻辑

### 10.3 查大叫（V1）

统一按以下规则实现：

- 未听牌者赔付听牌者 1 番底分

- 不要实现“最大可能番数赔付”。

### 10.4 一炮多响

必须支持：

- 一个点炮者对应多个胡牌者
- 每位胡牌者分别独立结算
- 点炮者承担各自赔付总和

### 10.5 撤销机制

必须支持：

- 撤销最近一条 settlement
- 删除后自动重新计算本局实时分数
- 若涉及血战状态，应同步回滚已胡玩家状态

## 11. 计分逻辑要求

所有计分逻辑必须集中在独立工具模块中，例如：

```test
/utils/score.ts
```

禁止在页面中直接实现核心计算。

### 基础公式

```TypeScript
totalFan = baseFan + genCount
effectiveFan = Math.min(totalFan, maxFan)
multiplier = Math.pow(2, effectiveFan - 1)
points = baseScore * multiplier
```

### 自摸

- 三家各付 points
- 胡牌者收 points * 3

### 点炮

- 点炮者付 points
- 胡牌者收 points
- 其他人为 0

### 杠牌

必须支持：

- 暗杠
- 明杠
- 加杠

杠牌结算也必须通过统一工具函数计算。
