# 四川麻将算番小程序 PRD

## 1. 产品定位

**一句话描述**：线下4人围坐四川麻将（血战到底/血流成河）的实时计分助手，单设备操作，零配置即用。

**核心场景**：牌桌旁一人手持手机，快速录入每局结算，实时展示4人累计盈亏，支持带根规则和一炮多响。

## 3. 功能模块

### 3.1 累计记分板（全局置顶）

| 元素 | 说明 |
|------|------|
| 显示位置 | 固定顶部，所有页面可见 |
| 内容 | 4玩家当前累计分数（可正可负） |
| 更新时机 | 每局点击[确认并累加]后刷新 |
| 数据持久化 | LocalStorage，关闭页面不丢失 |
| 重置方式 | 点击[重开]时询问是否清空累计 |

**交互细节**：
- 正分绿色显示，负分红色显示
- 点击玩家名可临时修改（仅当前会话）

---

### 3.2 规则配置模块

每局开始时选择，[下一局]时跳过此步骤。

| 配置项 | 选项 | 默认值 | 影响 |
|--------|------|--------|------|
| **游戏模式** | 血战到底 / 血流成河 | 血战到底 | 决定单局结束条件和结算次数上限 |
| **带根规则** | 开启 / 关闭 | 开启 | 开启后每根+1番，可叠加 |
| **封顶番数** | 3番 / 5番 / 8番 | 5番 | 计算上限，超过按封顶计 |
| **底分金额** | 1元 / 2元 / 5元 / 10元 | 1元 | 每番基数 |

**模式差异详解**：

| 特性 | 血战到底 | 血流成河 |
|------|---------|---------|
| 胡牌后处理 | 胡牌者退出，剩余继续 | 胡牌者继续摸打，可再次胡牌 |
| 单局结束条件 | 3人胡牌 或 流局查叫 | 牌墙打空（无牌可摸） |
| 结算次数 | 1-3次胡牌 + 多次杠牌 | 多次胡牌（同一人可多次）+ 多次杠牌 |
| 一炮多响 | 支持（剩余未胡者均可胡） | 支持（所有人均可胡） |

### 3.3 单局结算录入（核心模块）

根据当前模式动态渲染界面。

#### 3.3.1 血战到底模式

**状态指示器**：
本局进度：●玩家A(已胡)  ○玩家B  ○玩家C  ○玩家D  [剩余3家]

**操作按钮**：
- [胡牌登记] [杠牌登记] [查大叫/流局]

**胡牌登记表单**：
| 字段 | 类型 | 规则 |
|------|------|------|
| 胡牌人 | 单选（剩余未胡玩家） | 已胡玩家不可选 |
| 胡牌方式 | 点炮 / 自摸 | 二选一 |
| 点炮人 | 单选（剩余未胡玩家，不含胡牌人） | 点炮时必填，自摸时隐藏 |
| 基础番数 | 1番/2番/3番/4番/5番/6番+ | 选择后显示对应倍数 |
| 根数 | 0根/1根/2根/3根+ | 带根开启时显示，每根+1番 |

**结算后流程**：
1. 计算本番得分并累加到本局临时分数
2. 胡牌人标记为"已胡"，移出后续选择
3. 检查剩余人数：≤1人时自动弹出[本局结束]

**杠牌登记（刮风下雨）**：
| 类型 | 得分规则 |
|------|---------|
| 暗杠 | 三家各付2倍底分，杠家收6倍 |
| 明杠 | 点杠者付2倍底分 |
| 加杠 | 三家各付1倍底分，杠家收3倍 |

**查大叫（流局）**：
- 触发条件：点击[查大叫]或牌墙剩余7张时提示
- 录入：选择听牌玩家，未听牌玩家自动赔付
- 赔付规则：未听牌者按最大可能番数赔给听牌者（或约定1番底分）

#### 3.3.2 血流成河模式

**状态指示器**：
本局进度：玩家A已胡3次 │ 牌墙剩余：12张 │ 已进行：15分钟

**操作按钮**：
- [胡牌登记] [杠牌登记] [牌墙打空-结束本局]

**胡牌登记表单**（与血战相同，但所有人始终可选）：

**一炮多响支持**：
- 点炮后，界面提示"一炮多响，请选择所有胡牌人"
- 支持同时选择多个胡牌人，分别录入番数
- 点炮人分别赔付每位胡牌人

**结束条件**：
- 手动点击[牌墙打空-结束本局]
- 可选：设置牌墙剩余N张时提醒

## 4. 番数计算规则

### 4.1 基础番种表

| 番种 | 番数 | 说明 |
|------|------|------|
| 平胡 | 1番 | 基本胡牌型 |
| 对对胡 | 2番 | 全为刻子 |
| 清一色 | 3番 | 单一花色 |
| 七对 | 3番 | 七个对子 |
| 龙七对 | 4番 | 七对含一个杠 |
| 清对 | 4番 | 清一色+对对胡 |
| 清七对 | 5番 | 清一色+七对 |
| 天胡 | 5番 | 庄家起手胡 |
| 地胡 | 5番 | 闲家第一张牌胡 |
| 杠上花 | +1番 | 杠牌后自摸 |
| 杠上炮 | +1番 | 杠牌后点炮 |
| 抢杠胡 | +1番 | 抢他人加杠胡 |
| 海底捞月 | +1番 | 最后一张牌自摸 |
| 妙手回春 | +1番 | 最后一张牌点炮胡 |

### 4.2 带根规则（开启时）

- 每有一根（杠牌），总番数+1
- 根数可叠加：2根+2番，3根+3番
- 根与基础番数合并后计算封顶

### 4.3 计算公式

```javascript
// 输入
baseFan:      基础番数（番种表）
genCount:     根数（带根开启时）
maxFan:       封顶番数
baseScore:    底分金额
isZimo:       是否自摸

// 计算
totalFan = baseFan + genCount                    // 总番数
effectiveFan = min(totalFan, maxFan)             // 封顶处理
multiplier = 2^(effectiveFan - 1)                // 倍数：1番=1, 2番=2, 3番=4...
points = baseScore * multiplier                  // 单家应付金额

// 分配
if (isZimo) {
    // 自摸：三家付，赢家收3倍
    winnerGets = points * 3
    othersPay = -points each
} else {
    // 点炮：点炮者付，赢家收
    winnerGets = points
    loserPays = -points
    others = 0
}

**操作按钮**：
- [确认并累加] → 更新累计记分板 → 进入下一步选择
- [返回修改] → 回到结算录入，可增删记录

## 5. 数据模型

### 5.1 全局状态（LocalStorage）
```TypeScript
interface AppState {
    // 累计总榜（持久化）
    totalScores: {
        [playerName: string]: number  // 累计盈亏
    };
    
    // 玩家名称（持久化）
    playerNames: [string, string, string, string];  // 固定4人
    
    // 当前单局（临时，结束后可选保留或清空）
    currentRound?: {
        config: RoundConfig;
        settlements: Settlement[];
        tempScores: { [playerName: string]: number };  // 本局累计
        status: 'ongoing' | 'completed';
    };
}

interface RoundConfig {
    mode: 'blood_battle' | 'blood_river';  // 血战/血流
    enableGen: boolean;                     // 带根开关
    maxFan: 3 | 5 | 8;                      // 封顶
    baseScore: 1 | 2 | 5 | 10;              // 底分
}

interface Settlement {
    id: string;           // 时间戳
    type: 'hu' | 'gang' | 'chadajiao';
    timestamp: number;
    
    // 胡牌特有
    winner?: string;      // 胡牌人/杠牌人
    loser?: string;       // 点炮人（自摸为null）
    isZimo?: boolean;
    baseFan?: number;     // 基础番
    genCount?: number;    // 根数
    totalFan?: number;    // 总番（含根）
    
    // 杠牌特有
    gangType?: 'an' | 'ming' | 'jia';  // 暗杠/明杠/加杠
    
    // 计算结果
    points: number;       // 基础分
    distribution: {      // 4人实际得失
        [playerName: string]: number;
    };
}
```

## 7. 异常与边界处理

| 场景           | 处理策略                                  |
| ------------ | ------------------------------------- |
| **一炮多响**     | 点炮后界面提示"是否有多人胡牌"，支持添加多个胡牌记录，点炮人分别赔付   |
| **血战中途退出**   | LocalStorage保存`currentRound`，返回后恢复状态  |
| **误操作撤销**    | 单局内支持\[撤销上一步]，删除最近一条`settlement`并重新计算 |
| **提交后反悔**    | 累计记分板显示\[撤销本局]按钮，仅限下一局开始前             |
| **牌墙计数（血流）** | 手动输入剩余张数，或设置"已摸牌数"自动计算，到0时提示结束        |
| **分数溢出**     | 使用JavaScript Number安全范围，界面限制显示±99999  |
| **同名玩家**     | 初始化时检查并强制区分，支持临时修改昵称                  |
| **网络离线**     | 纯前端运行，无需网络，数据存LocalStorage            |
